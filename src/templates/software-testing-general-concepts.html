ISO 9000

  • ISO 9000 is a set of standards for quality assurance systems.
  • ISO 9000 defines Software Testing as:
    1. The process of exercising software to verify that it satisfies specified requirements,
    2. The standard procedure to detect errors.

Failure, Fault, and Error

Official IEEE terminology

  • Failure: an observable, incorrect behavior as compared to expected results.
  • Fault (aka BUG) is a defect within the system (software/hardware). May stay dormant for a long time before it manifests itself as an error. (segmentation fault)
  • Error: error is a mechanism by which the fault becomes apparent.

IEEE - Istituto degli ingegneri elettrici ed elettronici comportamento incorretto.
All physical components are subject to failure all systems are subject to random faults. Failure: Qualsiasi deviazione del comportamento osservato dal comportamento specificato. Fault exmaple: 1. Out of memory error. The programmer was not careful and consumed more memory than the hardware has available. 2. Segmentation fault. The programmer attempts to point to a portion of memory that is not in its "space". For example, an pointer to a memory block pointing into memory space owned and used by the operating system.

Random Testing

  • Pick inputs randomly, in the input domain.
  • Random testing eliminates designer bias.
  • Imagine an infinite grid. It is very unlikely by picking randomly to get some failure.
  • Fortunately not all is lost.
12-Black-Box Testing Videos

Failures are generally sparse and dense in some sub-domains

Correct approach

  • Identify partitions of our domain.
  • Select inputs from each partition (sub-domain).
  • We can dramatically increase our chances to reveal faults in the code.

Inspections - reviews walkthroughs

Examining the Code 1/2

  • Peer Reviews. The team reviews the code together and looks for problems and oversights.
  • Coding Standards and Guidelines. It's equivalent to writing words that can be understood but don't meet the grammatical rules of the English language.

Examining the Code 2/2

  • Data Reference Errors. Caused by using a variable that has not been properly declared or initialized.
  • Computation Errors. Errors which are essentially bad math.
  • Comparison Errors. Less than, greater than, equal, not equal, true, false.
  • Control Flow Errors. Is it possible that a loop never executes?
  • Subroutine Parameter Errors. Subroutine parameter errors are due to incorrect passing of data to and from software subroutines.
