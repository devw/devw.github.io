Introduction 3/2

  • Efficiency is the (often measurable) ability to avoid wasting materials, energy, efforts, money, and time in doing something or in producing a desired result.
  • In more mathematical or scientific terms, it is the capability of a specific application to produce a specific outcome with a minimum amount or quantity of waste, expense, or unnecessary effort.

What is efficiency?

  • Efficiency is not about code details (changing some instructions). 
    a = x ? x : y;
// The above expression is equivalent
// to the below expression
a = x ?: y;
// Clever but bad because
// it takes time to figure out.
  • Efficiency is about choosing the right algorithm.
  • Clever algorithms are hard to invent.
  • We want to find an algorithm that some clever people already know, in order to solve our problem.
How come I transform my problem to match a problem that some clever person already knows how to solve?

How do we think about efficiency?

  • We think about efficiency in two dimensions: space and time.
  • We can often trade one for the other.
  • We are going to focus our attention on time.
We can make a program run faster by using more memory or use less memory at the cost of making it run more slowly.

Time efficiency

  • How long do some algorithms take to run?
  • How would you answer this question?
  • I could run an algorithm on a computer based on some input and I could time it with my watch.
  • That would be a really bad way to answer.

Why?

  • The time is influenced by the speed of the machine.
  • It depends on the cleverness of the compiler implementation.
  • It depends on the input.

I need to talk about it in a abstract way

    We define a mathematical function where:
    1. the first natural number corresponds to the size of the input,
    2. the result of the function is the number of steps that the computation will take for an input of that size:
    T: N -> N

What is a step?

We are going to use a model of computer:

  • Random-access machine (RAM).
  • A step is an operation that takes constant time.

How long will an algorithm take to run?

There are several different ways we could answer (linear search):

  • Best case
  • Average case
  • Worst case (Murphy's law)

What's the worst that can happen? And that's nice because it means that there are no surprises.

Example 1/2

        var f = function(n) {
            var answer;
            answer = 1;
            while (n > 1) {
                answer *= n;
                n -= 1;
            }
            return answer;
        }
  • How long will this program take to run in terms of number of steps?
  • T(n) = 2 + 3n + 1

Example 2/2

var fib = function (n) {
    if (n === 0 || n === 1) {
        return 1;
    } else {
        return fib(n - 1) + fib(n - 2);
    }
}

  • How long will this program take to run in terms of number of steps?
  • T(n) = 2n

Big O notation

  • What is the function that characterizes the algorithm implemented by the previous code?
    1. When I look at function complexity I do ignore additive constants.
    2. We use a model of asymptotic growth to understand how the complexity increases.
    3. This is typically done by a notation called "big O notation".
    4. Example: T(n) = 2 + 3n + 1 -> O(n) (the time linearly increases with n)
